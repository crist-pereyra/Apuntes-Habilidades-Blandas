## Artefactos Scrum
### Backlog del producto
El la lista de todo lo que se conoce que es necesario en el producto, cualquier cambio debe agregarse a la lista, si no esta es que no exite.

Es dinamica para que sea util y competitiva (pueden agregarse o eliminarse los item de la lista)

Los elementos de la lista de un producto se denominan *Historias de Usuario*.
Todas las historias de usuario se pueden agrupar en elementos mas grandes denominados *Epicas*.

#### Epicas
Se crean a partir de funcionalidades del producto que se esta desarrollando. Son los modulos donde se desprenden cada grupo de caracteristicas del programa.

#### Historias de usuario
No son requerimientos, es lo que te esta contando el usuario
* **Titulo** *Subir tareas*
* **Descripicion** *Como se debe subir la tarea, arquitectura, BBDD, etc*
* **Puntos** *Esfuerzo que demora en completarla* estos deben ser evaluados de acuerdo a la complejidad, cantidad de trabajo requerido, conocimientos necesarios y la incertidumbre (factores externos) se puede utilizar el *Poker de planeacion*
* **Criterio de aceptacion** *Que se necesita para evaluar que la tarea esta completada*

#### Lista de pendientes del sprint
Es un subconjunto del la lista del Producto, estas seran desarrolladas durante el sprint
|HIstorias|Historias decididas para trabajar|Historias que se estan trabajando|Historias que se estan probando o evaluando|Historias que estan completas|
|----|----|----|----|----|

El equipo de desarrollo decide quitar los elemnetos de la lista de pendientes
Las prioridades de los elementos se basaran a partir de:
* Valor para el cliente
* Urgencia
* Riesgo / Oportunidad
* Esfuerzo
